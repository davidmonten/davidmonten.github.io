How it works

Files

When you compile a program you pick informations from different sources, merge it together and translate in binary code to let the micro controller to read it.

In order to be compiled every program in C+ needs:
a File.c: it contains the informations about the algorithms you want the micro has to do.
some Dependencies: “pieces” of program your File.c refers to; some of their functions will be embeeded in the compiled file. and
a File.make: it contains the “recepie” to “bake” the file that we want to compile, or the info to retrieve all the files (File.c and dependencies) to compile.

Keep all the files in the same folder can be a smart choice to start avoiding any sort of issue.

Microcontroller

I’ve used an ATtiny44.
Here you can find the datasheet, a detailed document about this device.

Its main parts are: Memory
there are three different kind of memory: flash, RAM and EEPROM; they are all subdivided in registers of 8 binary characters.

Clock It sets the amount of cycle per second; the built-in is works at 8MHz, but you can overide it with an external clock.

Timers It’s a counter and it counts events; it’s properly a timer when paired up with clock to have a time reference.

Interrupts They stop the program that is running to do something else, than the main program restarts from where it was stopped; is cames in play, for example, when a button is pressed.

Input/Output Ports ATtiny44 has group of 8 pins due to its 8bit structure. Every pin can be setted as input, output or none setting to 1 or 0 or floating the relative pin in the proper register.

Universal Serial Interface (USI) It’s the main way it communicates with other devices.
